# pi-fff — FFF-Powered Fuzzy Find & Grep Extension for pi

**Source:** https://github.com/SamuelLHuber/pi-fff
**Date:** 2026-04-10
**Tags:** developer-tools, fuzzy-search, TypeScript, rust, cli-extension

---

**What it is:** A [pi](https://github.com/badlogic/pi-mono) extension that swaps the built-in `find` and `grep` tools for [FFF](https://github.com/dmtrKovalenko/fff.nvim) — a Rust/SIMD-native search library with no subprocess overhead.

**Key upgrades over defaults:**

| Built-in | pi-fff | Win |
|---|---|---|
| `find` → spawns `fd` | `find` → FFF native | Fuzzy + frecency-ranked |
| `grep` → spawns `rg` | `grep` → FFF native | SIMD + mmap, no subprocess |
| *(none)* | `multi_grep` | OR-logic via Aho-Corasick |

**Standout features:**
- **No subprocess spawning** — FFF is a Node-bound Rust lib, calls are in-process
- **Pre-indexed** at session start → instant results
- **Frecency ranking** — learns which files you actually open, boosts them
- **Git-aware** — modified/staged files surface higher
- **Query history** — combo boost when a file was previously selected for a similar query
- **Cursor pagination** — grep results are pageable

**Install:**
```bash
pi install git:github.com/SamuelLHuber/pi-fff        # global
pi install -l git:github.com/SamuelLHuber/pi-fff     # project-local
pi install git:github.com/SamuelLHuber/pi-fff@v0.2.3 # pinned
```

**Relevance for you:** If you're using `pi` in any of your projects (lyra, voiceCLI, etc.), this would make file/content search noticeably snappier — especially in large codebases where `fd`/`rg` subprocess startup latency adds up. The frecency learning is a nice bonus for navigating repos you live in daily.
