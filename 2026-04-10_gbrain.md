# GBrain — Markdown-based personal knowledge brain with Postgres + pgvector

**Source:** https://github.com/garrytan/gbrain
**Date:** 2026-04-10
**Tags:** personal knowledge management, AI agent, markdown, vector search, OpenClaw

---

**GBrain** is an open-source TypeScript project by Garry Tan that turns a git repo of markdown files into a queryable personal knowledge base — the "memex" Vannevar Bush envisioned.

**Core idea:**
- One markdown file per person, company, idea, meeting, etc.
- Compiled facts at the top + append-only timeline at the bottom
- An AI agent enriches it continuously (including overnight via a "dream cycle")

**Scale it's designed for:**
| Data type | Volume |
|---|---|
| Markdown files | 10,000+ |
| People dossiers | 3,000+ |
| Calendar events | 21,000+ (13 years) |
| Apple Notes | 5,800+ |
| Meeting transcripts | 280+ |
| Original ideas | 300+ |

**Architecture:**
- `Markdown files` → source of truth (no DB required to start)
- `Postgres + pgvector` → hybrid keyword + semantic search at scale (needed past ~1,000 files)
- `CLI + MCP layer` → thin interface optimized for OpenClaw / Hermes agents

**Relevance to your stack:**
This maps closely to your **2ndBrain / roxabi-vault** work — same problem space (personal knowledge + agent retrieval), different implementation (Markdown + pgvector vs. SQLite + FTS5). The "dream cycle" pattern (nightly cron enriching memory) is worth stealing — your Lyra agent could run something similar against roxabi-vault entries overnight.
