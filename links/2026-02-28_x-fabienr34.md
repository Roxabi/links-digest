Vérifiez votre robots.txt. Anthropic met à jour la documentation des crawlers de Claude.

Si vous gérez du SEO technique, c’est important.

Anthropic définit désormais 3 bots distincts :

• ClaudeBot → entraînement du modèle
• Claude-User → récupération en temps réel lors des requêtes utilisateurs
• Claude-SearchBot → indexation pour les réponses de recherche de Claude

Chaque bot possède son propre contrôle indépendant via robots.txt.

Bloquer ClaudeBot ≠ bloquer Claude-User ou Claude-SearchBot.

On passe officiellement à une architecture segmentée :

• Crawlers d’entraînement
• Bots d’indexation / search
• Bots de récupération déclenchés par l’utilisateur

Ce que ça change :

Ce n’est plus :
“Autoriser l’IA” ou “Bloquer l’IA”

C’est :
✔ Bloquer l’entraînement
✔ Autoriser la visibilité dans la recherche
✔ Autoriser le live fetch
✔ Ou combiner selon votre stratégie

Pour rappel :

Cloudflare avait introduit des signaux de contrôle IA (search / ai-input / ai-train) côté infrastructure.

Ici, le changement vient directement du côté LLM.

OpenAI, Perplexity et Google fonctionnent déjà avec des architectures séparées.
Claude s’aligne désormais sur ce modèle.

Si vous gérez des politiques robots.txt,
il est temps de les auditer.

Vous bloquez peut-être involontairement votre visibilité, vos citations ou votre présence dans les réponses IA.

La visibilité IA devient une décision de configuration.
