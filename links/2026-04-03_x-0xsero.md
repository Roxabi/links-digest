How to set up Factory Droid with any model

Bring your own keys, route through a local proxy, or point at a self-hosted endpoint.

Droid lets you swap in any LLM that speaks the Anthropic, Responses, or OpenAI API format. You edit two JSON files, restart Droid, and your models show up in the model picker. That's it.

This guide will show you how to bring your own subscriptions into Droid, I have been using this feature for over 6 months now, and it's been my favourite harness sine then.

Provider Types

There are three values for provider, and each one changes how Droid talks to the API.

"anthropic"

For any API that uses the Anthropic Messages format. The base URL should NOT end in /v1.

"base_url": "https://api.anthropic.com"   // or any anthropic compatible api
"provider": "anthropic"

Works with: Claude Max Subscription, any proxy that speaks the Anthropic format (MiniMax, Kimi, ZAI's anthropic endpoint).

"openai"

For native OpenAI endpoints. The base URL should end in /v1.

"base_url": "https://api.openai.com/v1", // or any responses compatible api
"provider": "openai"

Works with: OpenAI, GPT models through a local proxy.

"generic-chat-completion-api"

The catch-all. Use this for any OpenAI-compatible endpoint that isn't actually Responses. Same /v1 convention.

"base_url": "https://api.example.com/v1",
"provider": "generic-chat-completion-api"

Works with: self-hosted models, local AI endpoints, Gemini proxies, Cursor proxy, anything that looks like the OpenAI Chat Completions API.

Using your subscriptions with VibeProxy

Install vibeproxy https://github.com/automazeio/vibeproxy

Trigger OpenAI Plus/Pro, Claude Pro/Max, Github Copilot oauth

Vibeproxy catches the oauth keys, and exposes an endpoint

Put endpoint (localhost:8317 for example) as you provider url

If you run a local proxy like VibeProxy, you can route multiple providers through a single gateway. The proxy handles auth, so you don't put real API keys in the config.

Common mistakes

Forgetting /v1 on OpenAI/generic base URLs, or adding it to Anthropic ones

Setting supports_images: true on a model that doesn't support vision (causes 400 errors when you paste a screenshot)

Leaving api_key empty instead of using a vibeproxy configure password for using your subs in Droid

Getting the model ID wrong. The model field has to match what the API actually expects, not what you want to call it. That's what model_display_name is for
