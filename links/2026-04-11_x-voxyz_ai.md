telegram just opened bot-to-bot communication. bots can trigger each other in groups through replies and @mentions.

discord could do this, but heavy UI. telegram is clean and fast, but bots couldn't see each other. now they can.

→ BotFather setup
/mybots → select bot → Bot Settings → Bot-to-Bot Communication → Enable

→ if you're using openclaw for QA, three env vars
OPENCLAW_QA_TELEGRAM_GROUP_ID=your private group
OPENCLAW_QA_TELEGRAM_DRIVER_BOT_TOKEN=the bot sending commands
OPENCLAW_QA_TELEGRAM_SUT_BOT_TOKEN=the bot being tested

→ run it
openclaw qa telegram

openclaw verifies reply threading, reactions, and /help command automatically. outputs a report.

→ one big trap: infinite loops
two bots replying to each other can go forever. telegram docs explicitly warn about this. you need dedup, rate limiting, and max interaction depth.
