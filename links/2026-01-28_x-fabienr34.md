Ollama vient de lâcher un game changer : ollama launch

Une seule commande.
Zéro config.
Des agents IA prêts à l’emploi.

👉 Claude Code
👉 Codex
👉 OpenCode
👉 Clawdbot
👉 Droid

Le tout en local ou via le cloud.

🧠 Jusqu’ici, lancer un agent IA, c’était souvent :

❌ variables d’environnement

❌ fichiers de config
❌ clés API partout
❌ setup fragile
Avec ollama launch, tout disparaît.

Tu lances.

Ça fonctionne.
Point.

⚙️ Ce que fait ollama launch

Il configure automatiquement :

le modèle

le runtime
les connexions
le contexte
le mode local ou cloud
👉 aucune variable à définir
👉 aucun fichier .env

👉 aucune prise de tête
C’est littéralement le “docker run” des agents IA.

Modèles disponibles

GLM 4.7 Flash (local) : ~23GB de VRAM
jusqu’à 64k tokens de contexte

la commande : ollama pull glm-4.7-flash

GLM 4.7 Cloud

contexte complet
géré par le cloud Ollama
free tier généreux

la commande : ollama pull glm-4.7:cloud

Tu choisis :
💻 local
☁️ cloud

Lancer les agents

Claude Code : ollama launch claude

Codex : ollama launch codex

OpenCode : ollama launch opencode

Clawdbot : ollama launch clawdbot

Chaque agent démarre avec le bon modèle, automatiquement.

Pourquoi c’est énorme ?

Parce qu’on change de paradigme.

On ne “configure” plus l’IA.
On la lance.

Comme un service système.
Comme un outil natif.
Comme un vrai copilote.
