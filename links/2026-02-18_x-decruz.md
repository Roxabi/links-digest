Mine isn't that complex. I use nested CLAUDE/AGENTS.md for my major code folders. Then progressive disclosure via the root CLAUDE file. This feels very useful to me to make my workflow when it comes to file exploration.

I have progressive disclosures for docs/learnings, where mistakes and rectification is stored, along with learnings from the session.

I have a docs/golden_rules, which stores the golden rules which my agent needs to follow when coding. Additionally I codify some of those as part of my ESLint config, which runs the linter as a PostToolUse Hook.
