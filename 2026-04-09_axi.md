# AXI — Agent eXperience Interface Framework

**Source:** https://axi.md/
**Date:** 2026-04-09
**Tags:** CLI, agent-ergonomic, token optimization, developer tools

---

AXI (Agent eXperience Interface) is a framework of "10 design principles for agent-ergonomic CLI" tools. It includes reference implementations for GitHub operations (`gh-axi`) and browser automation (`chrome-devtools-axi`).

### Main Features
- **TOON output format** for "approximately 40% token savings over equivalent JSON"
- **Combined operations** that "eliminate the separate snapshot calls that MCP conditions require"
- **Pre-computed aggregates** and contextual next-step suggestions
- **Minimal default schemas** (3–4 fields) with optional expansion
- **Shell composability** via pipes ("`| grep`, `| head`, `| tail`")

### Target Audience
Developers building CLI tools intended for AI agent use. The framework addresses the specific constraint that agents operate within token budgets. As the site states: "Let's build apps that [agents] love."

### Key Result
In benchmarks across 915 runs, AXI achieved "100% task success at the lowest cost" ($0.074/task) compared to both raw CLI and MCP alternatives.
