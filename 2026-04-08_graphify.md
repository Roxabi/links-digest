# Graphify — Code-to-Knowledge-Graph AI Skill

**Source:** https://github.com/safishamsi/graphify
**Date:** 2026-04-08
**Tags:** knowledge graph, code analysis, tree-sitter, multimodal, AI skill

---

An AI coding assistant skill that transforms folders of code, documents, papers, images, videos, and URLs into a queryable knowledge graph. Works with multiple platforms including Claude Code, Codex, Cursor, Gemini CLI, GitHub Copilot CLI, and others.

### Main Features
- **Three-pass extraction:** AST parsing for code (no LLM), local Whisper transcription for audio/video, parallel subagent extraction for docs/images
- **Multimodal support:** Code (20 languages via tree-sitter), PDFs, markdown, images, video, audio, YouTube links
- **Graph outputs:** Interactive HTML visualization, queryable JSON, audit report (GRAPH_REPORT.md)
- **Confidence transparency:** Every edge tagged as EXTRACTED, INFERRED (with confidence score), or AMBIGUOUS
- **71.5x token reduction** versus reading raw files on large corpora
- **Always-on integration:** Platform-specific hooks/instructions that surface the graph before file searches
- **Git hooks:** Auto-rebuild on commit and branch switch
- **MCP server support:** Structured graph queries via `query_graph`, `get_node`, `get_neighbors`, `shortest_path`

### Tech Stack
NetworkX + Leiden (graspologic) + tree-sitter + vis.js. Runs entirely locally; no Neo4j or vector database required. Semantic extraction uses your AI platform's underlying model API.

### Use Cases
- Quickly understand unfamiliar codebases by navigating structure instead of grepping files
- Manage "raw" folders of papers, notes, and screenshots as queryable knowledge
- Trace architectural rationale and design decisions across code and documentation
- Build persistent graphs that remain useful across sessions without re-processing
